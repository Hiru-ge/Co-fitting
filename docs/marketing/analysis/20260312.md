### 最重要発見

3. **Qiita記事が最大の流入源**（全体の75%）。3/8に300人の大量流入があり、以降は減衰中
4. **訪問記録まで到達したユーザー（5人）はヘビーユーザー化**。バッジ獲得・レベルアップまで使い込んでいる

### 3.2 ファネル各ステップの詳細

#### Step 1: LP → beta-gate（合言葉入力）

| 指標 | 値 |
|------|-----|
| LP到達ユーザー | 467人 |
| beta-gate到達ユーザー | 118人 |
| **コンバージョン率** | **25.3%** |
| beta-gateの平均セッション時間 | 71秒 |

- LPを見た467人のうち、**74.7%がbeta-gateに到達する前に離脱**
- ただしbeta-gateが「壁」として機能するのは意図的（ベータ限定のため）
- beta-gateのバウンス率は1.6%と低く、到達した人はほぼ合言葉を入力しようとしている

#### Step 2: beta-gate → ログイン（Google OAuth）

| 指標 | 値 |
|------|-----|
| beta-gate通過 → login画面 | 25人 |
| login イベント発火 | 21人（ユニーク） / 46回（延べ） |
| **コンバージョン率** | **17.8%**（beta-gateからloginまで） |
| form_start（合言葉入力開始） | 47人 |

- **最大のボトルネック**。beta-gate到達118人のうち、ログインまで到達したのは21人
- form_start が47人なので、**合言葉を入力しようとしたのは47人**。うち半数程度が合言葉を知らず離脱したと推定
- 合言葉を知っていてもGoogle OAuthのハードルで一部が離脱している可能性

#### Step 3: ログイン → オンボーディング

| 指標 | 値 |
|------|-----|
| ログインユーザー | 21人 |
| オンボーディング完了 | 17人 |
| オンボーディングスキップ | 2人 |
| **コンバージョン率** | **90.5%** ✅ |

- オンボーディング（興味タグ選択）の完了率は極めて高い
- スキップしたのは2人のみ。興味タグ設定UIの導線は良好

#### Step 4: 提案生成 → 提案閲覧 → 訪問記録

| 指標 | 値 |
|------|-----|
| 提案生成（suggestion_generated） | 82回 / 21人 |
| 提案閲覧（suggestion_viewed） | 779回 / 21人 |
| 提案スキップ（suggestion_skipped） | 547回 / 19人 |
| 提案リロード（suggestion_reloaded） | 40回 / 14人 |
| **訪問記録（visit_recorded）** | **16回 / 5人** |

**分析:**
- 1人あたり平均 **3.9回** 提案を生成し、**37.1回** 提案カードを閲覧
- **提案スキップ率が高い**（547回スキップ / 779回閲覧 = **70.2%**）。提案の質に課題がある可能性
- 提案リロード（残り回数を消費して新しい提案を取得）は14人が利用。**「もっと良い提案が欲しい」というニーズ**がある
- **訪問記録への転換率は23.8%**（21人中5人）。提案を見ても実際に行く人は少数

#### Step 5: ゲーミフィケーション到達

| イベント | 回数 | ユーザー |
|---------|------|---------|
| バッジ獲得 | 12回 | 5人 |
| レベルアップ | 8回 | 4人 |
| デイリー完了 | 2回 | 2人 |
| メモ保存 | 1回 | 1人 |

- 訪問記録した5人は**全員バッジを獲得**。ゲーミフィケーションの仕組みは機能している
- 1人あたり平均2.4個のバッジを獲得
- **メモ保存は1回のみ**。メモ機能の利用率が極めて低い
- デイリー完了も2人のみ。日次のリテンションメカニズムが弱い

## 7. 課題分析と改善提案

### 7.1 致命的課題（即対応）

#### 🔴 課題1: beta-gate → ログインの離脱率82%

**現状:** beta-gate到達118人 → ログイン21人（17.8%）  
**原因推定:**
- 合言葉を知らないユーザーが多数（form_start 47人 → ログイン21人 = 半数が合言葉不明で離脱）
- Google OAuthの心理的ハードル

**改善案:**
- ベータ期間を終了し、合言葉ゲートを撤廃（最も効果が大きい）
- 合言葉をLP上により目立つ形で掲示
- OAuthの代替としてメールログインの導入

#### 🔴 課題2: 提案 → 訪問記録の転換率が低い（23.8%）

**現状:** 提案閲覧21人 → 訪問記録5人  
**原因推定:**
- 提案スキップ率70.2%（提案の質/関連性に課題）
- 「提案を見る」と「実際に行く」の間に大きな心理的ギャップ
- 訪問記録のタイミング・UI・動機付けが不十分

**改善案:**
- 提案アルゴリズムの精度向上（興味タグとの一致率改善）
- 「今すぐ行ける近場」の提案モード追加
- 訪問記録のリマインダー通知

### 7.2 重要課題（中期対応）

#### 🟡 課題3: Qiita依存の流入構造

**現状:** 流入の75%がQiita。記事バズ後の急速な減衰  
**改善案:**
- SEO対策（サイトマップ提出、メタ情報最適化、ブログコンテンツ追加）
- SNS戦略の強化（現状X/Instagramのバウンス率16.7%は高い）
- ASO（App Store Optimization風のPWA検索対策）

#### 🟡 課題4: メモ・デイリー機能の利用率が極端に低い

**現状:** メモ保存1回、デイリー完了2回  
**原因推定:**
- メモ保存の導線が弱い（訪問記録後にメモ入力を促すUIがない？）
- デイリー機能の存在に気づいていないユーザーが多い

**改善案:**
- 訪問記録直後にメモ入力を促すモーダル/フロー
- デイリーミッションのプッシュ通知

#### 🟡 課題5: 履歴ページのエンゲージメントが低い

**現状:** `/history` の平均セッション時間14秒（全ページ最短）  
**改善案:**
- 訪問履歴の可視化強化（マップビュー、統計サマリー）
- 過去の訪問の振り返り・共有機能

### 7.3 ポジティブな発見

#### ✅ 良い点1: バウンス率が全体的に非常に低い（3.1%）

LPのバウンス率1.3%は非常に優秀。LPのコンテンツ・デザインがユーザーの興味を引いている。

#### ✅ 良い点2: リピーターのエンゲージメントが極めて高い

リピーター45人の平均セッション時間342秒（5.7分）、PV/セッション8.7。一度使い込むとハマるプロダクト。

#### ✅ 良い点3: オンボーディング完了率90.5%

興味タグ設定のオンボーディングはスムーズに機能している。

#### ✅ 良い点4: ゲーミフィケーションの効果

訪問記録した5人は全員バッジ獲得・レベルアップまで到達。ゲーミフィケーションが「もう1回使おう」の動機になっている。

#### ✅ 良い点5: 提案閲覧の深さ

ログインユーザー21人が平均37回提案カードを閲覧。「提案を見ること自体」に価値を感じている。
