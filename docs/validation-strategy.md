# ニーズ検証・プレローンチ戦略

> 決定日: 2026-02-07
> 前提: プロダクト開発は確定。検証の目的は Go/No-Go 判断ではなく「方向性の微調整」と「ローンチ初速の確保」。

## 判断の背景

Makers Guild（第1章〜第3章、ミニマム公開記事）は、開発前に本格的な市場検証を推奨している。しかし Roamble は以下の条件を満たすため、フルセットの検証は不要と判断した。

- **ドッグフーディング前提のプロダクト**（Makers Guild 1.1「自分が本当に欲しいもの」に該当）
- **アイデア3条件スコアリング済み**（1.2: 18/25点 — 有望ライン達成）
- **競合分析済み**（要件定義書に4社比較あり）
- **Phase 0 自体が検証フェーズ**（友人5〜10名テスト → KPI測定）

## やること（3つ）

### 1. LP + メール収集（ウェイトリスト）

- **目的**: ローンチ初日にユーザーがゼロにならないようにする + 課題の言語化データ収集
- **根拠**: Makers Guild 1.5「LPテスト」の簡略版。有料広告は不要（ターゲットが日本語圏のためオーガニックで十分）
- **ツール**: Carrd / Framer / Notion Site（30分〜1時間で作成）
- **メール収集**: Tally or Google Forms
- **設問**: 「休日の過ごし方で一番困っていることは？」（自由記述1問のみ）
- **成功基準**: 登録者数は追わない。課題の言語化データが3件以上集まれば十分

### 2. X 共感投稿（2〜3本）

- **目的**: 課題フレーミングの検証 + 初期フォロワーの種まき
- **根拠**: Makers Guild 1.3「X検索」+ ミニマム公開記事「#BuildInPublic」の前段階
- **投稿方針**: アンケートではなく「共感を問う」投稿。バイアスの強い「使いますか？」系は避ける
- **投稿例**:
  - 「休日、結局いつもの店に行っちゃう現象に名前つけたい」
  - 「新しい場所に行きたいけど探すのがめんどくさい、って何なんだろう」
- **観察ポイント**: いいね/RT数（共感度）、リプの言葉遣い（LP コピーに活用）
- **成功基準**: 反応が薄ければ課題の言語化を変える。反応があればLP誘導に使う

### 3. X キーワード検索（5分）

- **目的**: 課題を語っている人が実在するか確認 + ユーザーの語彙を把握
- **根拠**: Makers Guild 1.3「ネガティブキーワード検索」の最小実行
- **検索ワード**: 「休日 マンネリ」「休日 同じ場所」「新しい場所 行きたい めんどくさい」「散歩 飽きた」
- **得られるもの**: ユーザーが使う言葉（LPコピーに直結）、見落とし競合の発見

## やらないこと（判断理由付き）

| アクション | Makers Guild 該当箇所 | やらない理由 |
|---|---|---|
| 簡易インタビュー5名 | 1.5 | Phase 0完成後に友人テスト（5〜10名）で代替。プロダクト有りのフィードバックの方が高品質 |
| Reddit投稿 | 1.3, ミニマム公開記事 | 初期ターゲットが日本人20-30代。英語圏Redditはターゲット不一致 |
| YouTube需要リサーチ | 1.3 | 動画需要とアプリ需要は別物。工数に見合わない |
| Product Hunt ティザー | ミニマム公開記事 | 動くプロダクトが無い段階では不適切。Phase 0完成後に実行 |
| 有料広告テスト | 1.5 | 日本語圏ターゲットへの英語圏広告は効率が悪い。オーガニックで十分 |
| アイデア評価シート再実施 | 1.2 | 要件定義書で18/25点として実施済み |
