## 🔧 Phase 1 仕上げ（ドッグフーディング）

実際にアプリを使って気になった点を随時追記・チェックしていく。

- [ ] 1日3提案までが機能していない。3施設目の完了をして「再試行」すると、また施設提案がされてしまう。
- [ ] 単に施設提案ができなかった時(不具合)と、3提案完了で提案が止まる場合のユーザーフィードバックが同じになっている。前者は「エラーが起きた」ことを伝える必要があり、後者は「コンプリート」であることを伝える必要がある。
- [ ] パーソナライズ提案が機能していない。どんな興味タグを設定していても同じ提案がされるので、原因特定と解決を試みる。
- [ ] ユーザー情報設定ページ内のアカウント削除は「ユーザ情報」タブ内に移して良い。現在は「アカウント」タブ内にあるが、Google人量のみにしたことで、各タブの内容が薄くなっているため、統合して迷いにくい配置にする。あと、「ユーザ情報」ではなく「ユーザー情報」という表記にする。
- [ ] ユーザー情報設定ページから、提案半径を変更できるようにする。現在は半径固定で5kmになっているが、ユーザーによってはもっと広い(狭い)範囲で提案してほしい可能性があるため。
- [ ] オンボーディング時に「スキップ」を押してもオンボーディングをスキップできない不具合がある。スキップを押すと、次の画面に遷移するのではなく、同じ画面に留まってしまう。これを修正する。
- [ ] バッジ獲得時の演出をもっと派手にする。今は経験値獲得モーダルの下にちょこんと表示されるだけだが、もっとユーザーが「やった！」と思えるような演出にする。例えば、獲得したバッジが画面中央に大きく表示されるようにするなど。トースト通知も出す。
- [ ] 発見ページ・提案設定ページで必要以上に下にスクロールできてしまう不具合がある。CSSでのheightの指定や、スクロールコンテナのoverflow設定を見直して、必要以上にスクロールできないようにする。
- [ ] マイページ画面の共有ボタンはまだ機能していないので、「ランキング」ボタン同様にグレーアウトして、まだ使えないことがユーザーにわかるようにする。
