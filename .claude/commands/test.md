# テスト実行

以下の順番でテストを実行し、結果を報告する。

## 1. バックエンド（Go）

```bash
cd backend && go test ./...
```

失敗したテストがある場合は `-v` オプションで詳細を確認する。

## 2. フロントエンド（Vitest）

```bash
cd frontend && npx vitest run
```

## 3. E2E テスト（オプション）

引数に `e2e` が含まれる場合、またはバックエンドのエンドポイント・認証・リダイレクトに変更があった場合は E2E テストも実行する。
E2E テストにはバックエンドコンテナが起動している必要がある（`docker compose ps` で確認）。

```bash
cd frontend && npx playwright test
```

## 完了条件

- すべてのテストが PASS していることを確認
- 失敗があれば原因を特定し修正案を提示する
- `docs/todo.md` の対象タスクにチェックを入れる
